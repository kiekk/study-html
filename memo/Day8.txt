Bos-Sizing
width, height 값에 padding이나 border 영역은 포함되지 않습니다.
따라서 width가 100%로 설정될 경우 padding이나 border 속성을 추가할 수 없습니다.
CSS3 부터는 box-sizing이라는 새로운 속성이 생겼습니다.
box-sizing을 설정하면 해당 요소의 패딩과 테두리가 포함되어 계산됩니다.
content-box : 기본값, padding과 border는 폭과 높이에 포함되지 않고 컨텐츠 영역을
기준으로 크기가 결정됩니다.
border-box : border를 기준으로 크기를 정합니다. content 영역의 폭과 높이는 padding과
border를 포함한 범위가 됩니다.

둥근 테두리 만들기
border-radius : 속성으로 테두리를 둥글게 만들 수 있습니다.
border-radius : top-left-x top-right-x bottom-right-x bottom-left- x /
	        top-left-y top-right-y bottom-right-y bottom-left-y 

테이블 만들기
HTML  요소 중 테이블 요소의 스타일을 설정할 수 있습니다.
스타일을 table 요소나, th, td 요소등에 주는 방법은 동일합니다.
하지만 테이블만 들어가는 border 관련 속성이 추가되어 있습니다.
border-collapse : 테이블 셀들간에 공백을 처리할지 여부를 결정하는 속성입니다.
이 속성은 tr, th, td 요소들에는 사용할 수 없습니다.
separate : 기본 값, 셀들을 분리시켜 사이의 공백을 만듭니다.
collapse : 셀들을 합쳐서 사이의 공백을 제거합니다. 셀이 합쳐지면서 border도 합쳐집니다.

vertical-align
테이블의 텍스트를 세로 정렬할 수 있습니다. 테이블 내용의 제목셀(th)안에 내용의
가로정렬은 기본이 가운데 정렬입니다.
셀(td)안에 내용 정렬은 기본 가로 정렬이 왼쪽 정렬이며, 세로 정렬은 가운데 정렬입니다.

CSS display
웹 페이지의 레이아웃을 결정하는 속성입니다.

1.블록 : display 속성값이 블록인 요소는 언제나 새로운 라인에서 시작하고, 해당 라인의
모든 너비를 차지합니다.
2.인라인 : display 속성값이 인라인(또는 인라인블록)인 요소는 새로운 라인에서 시작하지
않습니다. 또한 너비도 해당 라인 전체가 아닌 해당 HTML 요소의 내용만큼만 차지합니다.

인라인, 인라인블록의 차이
인라인에서는 할 수 없는 것들
1.  width/height를 적용할 수 없습니다.
2. margin / padding을 적용할 수 없습니다.
3. line-height를 적용할 수 없습니다.

display 속성의 기본 설정값의 변경
display 속성값이 블록인 요소의 속성값을 인라인(또는 인라인블록)으로 바꿀 수 있습니다.
또한 반대로 display 속성값이 인라인(또는 인라인블록)인 요소도 속성값을 블록으로 바꿀 수 있습니다.

display : none; 
웹 페이지에서 해당 요소가 나타나지 않고, 레이아웃에서 사라집니다.
레이아웃에서 공간을 차지하지 않습니다.
visibility : hidden;
해당 HTML요소를 웹 페이지에서 나타내지 않습니다.
레이아웃에서 공간을 차지합니다.

CSS 폼

height : 폼 요소의 높이를 설정합니다.

	또한 border-radius 속성을 이용하여 폼 요소의 모서리를 둥글게 만들 수 있습니다.
background-color : 폼 요소의 배경색을 설정합니다.
color : 텍스트의 색상을 설정합니다.
:focus : 폼 요소가 포커스를 가지고 있을 때 스타일을 설정할 수 있습니다.
background-image : 폼 요소에 아이콘이나 이미지를 삽입할 수 있습니다.또한
background-position 속성을 이용하여 삽입한 아이콘이나 이미지가 나타날 위치를
설정할 수 있습니다.
textarea
resize : textarea 요소의 크기를 조절할 수 있습니다. 해당 textarea 요소의 오른쪽
아래 부분에 마우스를 잡을 수 있는 핸들이 생깁니다. 
- none : 핸들이 사라지며 크기를 조절할 수 없습니다.
- both : 높이와 너비 모두 조절할 수 있습니다.
- horizontal : 너비만을 조절할 수 있습니다.
- vertical : 높이만을 조절할 수 있습니다.

CSS position
HTML 요소의 위치를 결정하는 방식을 설정합니다.

1. 정적 위치 지정 방식 (static position)
HTML 요소의 위치를 결정하는 기본적인 방식입니다.
position 속성값이 static으로 설정된 요소는 top, left, right, bottom 속성값에
영향을 받지 않습니다. 정적 위치 지정방식은 단순히 웹 페이지의 흐름에 따라
차례대로 요소들의 위치를 결정하는 방식입니다.
2. 상대 위치 지정 방식 (relative position)
HTML 요소의 기본 위치를 기준으로 위치를 재설정 하는 방식입니다.
HTML 요소의 기본 위치란 정적 위치 지정방식일 때 결정되는 위치를 의미합니다.
3. 고정 위치 지정 방식 (fixed position)
뷰포트를 기준으로 위치를 설정하는 방식입니다. 웹 페이지가 스크롤 되어도
고정 위치로 지정된 요소는 항상 같은 곳에 위치하게 됩니다.
뷰 포트 : 렌더링 되는 화면(실제 보는 화면), pc(브라우저의 화면) 모바일(전체화면)
브라우저들은 viewport로 아렬진 가상의 화면을 가지고 있습니다. 즉 화면 display상의
표시 영역을 말합니다. 데스크탑(pc)의 viewport는 브라우저 창의 viewport와 같고,
사용자가 브라우저 창의 크기를 조절하면 viewport의 크기도 조절됩니다.
반면 모바일 viewport는 상하 좌우로 스크롤을 움직이거나 줌인, 줌아웃을 통해
viewport를 변경하여 사용합니다.

4. 절대 위치 지정 방식 (absolute position)
뷰포트를 기준으로 위치를 결정하는 고정 위치 지정방식과 비슷하게 동작합니다.
단지 뷰포트만을 기준으로 하는 것이 아닌 조상요소를 기준으로 위치를 결정할 수 있습니다.
만약 위치가 설정된 조상요소를 가지지 않으면 body요소를 기준으로 결정하게 됩니다.

조상 요소라는 것은 정적 위치 지정방식을 제외한 방식(relative, fixed, absolute)로
위치가 설정된 조상요소를 의미합니다.
조상요소가 정적 위치 방식일 경우 위치는 뷰포트를 기준으로 결정하게 됩니다.

z-index
HTML 요소의 위치를 설정하게 되면 설정된 위치 및 방식에 따라 겹칠 수 있습니다.
겹쳐지는 요소들이 쌓이는 스택의 순서를 설정합니다. 스택의 순서는 양수나 음수
모두 사용할 수 있으며, 크기가 클수록 앞쪽에 위치하고 작을수록 뒤쪽에 위치하게 됩니다.

float
HTML 요소가 주변의 다른 요소들과 자연스럽게 어울리도록 만들어줍니다.
이 속성은 본래 다른 요소들과 어울리도록 위치하는 기능이 목적이지만 현재는 웹 페이지
레이아웃을 작성할 때 오히려 더 자주 사용됩니다.
1.해당 요소
left: 요소를 좌측으로 붙여줍니다.
right: 요소를 우측으로 붙여줍니다.
content 크기 만큼만 영역이 설정됩니다.
다른 요소보다 위쪽으로 위치됩니다.

2. 요소 다음으로 오는 요소들
설정한 방향 반대쪽으로 끌어올려 집니다.
