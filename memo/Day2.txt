HTML 태그
포트
문단 태그 : <p></p>
내용상 끊어서 구분할 수 있는 하나하나의 부분을 의미하고 <p>태그를 이용하여 이러한 문단을 
표현합니다. (문장이 끝나면 자동 줄바꿈)

블록 태그, 인라인 태그
블록 태그 : 입력된 데이터 영역뿐만 아니라 한 영역 자체를 차지하는 태그
인라인 태그 : 입력된 데이터의 해당 영역만 차지하는 태그

제목 태그 : <hn></hn> 헤딩 태그 ( n은 변수, 숫자)
HTML은 제목을 표현할 수 있는 다양한 크기의 <h>태그를 제공합니다.
가장 큰 <h1>태그 부터 가장 작은 <h6>태그까지의 크기로 제목을 표현할 수 있습니다.
검색엔진에 포함될 수 있는 태그

*검색엔진은 각 웹사이트의 내용을 <h>태그를 통해서 키워드를 수집합니다. 따라서 
HTML문서에 포함되는 제목은 <h>태그로 작성하면 검색엔진에 노출되는 확률을 높일 수 있습니다.

서식 태그 <strong>, <em>, <mark>, <del>, <ins>, <sup>, <sub>
<strong> : 텍스트를 굵게 표현하고 싶을 때 사용합니다. <b>와 표현이 동일하지만,
사용 용도가 다릅니다. <b>태그는 단순히 텍스트를 굵게 표시하는 역할만 하지만,
<strong>태그는 단순히 보여지는 강조가 아닌, 실제 페이지 내의 중요한 부분을 브라우저에게 
알려주는 역할을 합니다. (웹 접근성)

<em> : 텍스트를 이탤릭체로 표현하고 싶을 때 사용합니다. <i>와 표현이 동일하지만,
사용 용도가 다릅니다. <i>태그는 단순히 텍스트를 이탤릭체로 표시하는 역할만 하지만,
<em>태그는 텍스트를 이탤릭체로 변환하고 그 내용이 중요하다는 의미도 함께 포함 해줍니다.

<mark> : 텍스트 하이라이팅 효과를 적용시켜줍니다.

<del> : 텍스트 중앙에 가로줄을 만들어 텍스트를 지운 것 같은 효과를 줍니다.

<ins> : 텍스트 밑에 선을 그어 텍스트의 강조 효과를 줍니다.

<sup> : (superscript) 태그는 위첨자에 사용,
<sub> : (subscript) 태그는 아래첨자를 표현할 떄 사용합니다.

HTML문서의 구조
<!DOCTYPE HTML> : HTML문서의 선언이며, 버전 5를 나타냅니다.
<html lang="en"> : lang 속성은 웹 접근성에 관한 명시입니다. 스크린 리더에서 언어를 인식하여
	자동으로 음성을 변환하거나, 해당 언어에 적합한 발음을 제공할 수 있도록합니다.
<meta charset="UTF-8"> : charset 속성은 문자를 인코딩시에 문자코드를 설정하는 역할을
	지정합니다. 한국어, 영어 등만 사용할 경우 윈도우의 기본 문자셋인   EUC-KR 문자셋을
	사용해도 되지만, 다국어로 인코딩 하기 위해서는 UTF-8 문자셋을 사용하길 권장합니다.
<meta name="viewport" content="width=device-width, initial-scale=1.0">
: 모바일로 최적화된 사이트에 포함하는 META 속성입니다.


HTML문서의 속성
속성은 해당 요소에 대한 추가적인 정보를 제공합니다.
            ↓ 웹 접근성에 관련한 속성
<html lang = "en">
------ -----   ------
   요소  속성    속성값

* 속성의 값은 따옴표를 꼭 사용하지 않아도 되지만, 속성값에 띄어쓰기가 있을 경우 문제가
발생할 수 있으므로, 땅모표를 쓰기 권장합니다. (따옴표, 쌍따옴표 둘다 가능)

<img src = apple.jpg> ( O )
<img src = 맛있는 사과.jpg> ( X ) -> 맛있는 까지만 속성값으로 인식합니다.
<imt src = "맛있는 사과.jpg"> ( O )
<imt src = '맛있는 사과.jpg'> ( O )

목록 태그 : <ul>, <ol>, <dt>

1. 순서가 없는 목록 태그 
<ul> 태그로 시작하며, 포함하는 목록의 요소는 <li>태그로 시작합니다.
· 김사과
· 오렌지
· 바나나

<ul>
   <li>김사과
   <li>오렌지
   <li>바나나
</ul>
2. 순서가 있는 목록 태그
<ol> 태그로 시작하며, 각 목록 요소는 <li> 태그로 시작합니다.
1. 김사과
2. 오렌지
3. 바나나

<ol>
   <li>김사과
   <li>오렌지
   <li>바나나
</ol>

*type 속성의 값
1 : 숫자로 순서를 설정합니다. <ol type ="1">  -> 1, 2, 3...
A : 알파벳 대문자로 순서를 설정합니다. <ol type ="A"> -> A, B, C...
a : 알파벳 소문자로 순서를 설정합니다. <ol type ="a"> -> a, b, c...
I : 로마 대문자로 순서를 설정합니다. <ol type = "I"> -> I, II, III...
i : 로마 소문자로 순서를 설정합니다. <ol type = "i"> -> i, ii, iii....

*start 속성의 값
시작값을 임의로 설정할 때 사용합니다.
<ol type ="A" start = "3"> -> C,D,E...순으로 시작값이 C로 변경

<dt>태그에는 용어 이름을, <dd>태그에는 용어에 대한 정의가 들어갑니다.

이미지 <img>
비트맵 이미지, 벡터 이미지 차이
비트맵 이미지 : 비트맵은 각 픽셀이모여 만들어진 정보의 집합으로 레스터 이미지라고도 부릅니다.
		픽셀 단위로 화면에 렌더링합니다. 그림판, 포토샾등으로 편집할 수 있습니다.
벡터 이미지 : 수학적 정보의 형태들이 만들어내는 결과물입니다. 이미지가 가지고 있는 점, 선, 면의 
		위치, 색상 등의 정보를 가지고 있으며 그를 화면에 렌더링합니다.
		확대 및 축소를 해도 이미지가 깨지지 않습니다. 일러스트로 편집할 수 있습니다.
 JPG(JPEG)
압축률이 훌륭하여 사진이나 예술분야에서 많이 사용됩니다.
- 손실 압축
- 표현 색상(24bit, 약 1600만 색상)
- 이미지의 품질과 용량을 쉽게 조절 가능
- 가장 널리 쓰이는 이미지 포맷

GIF
이미지 파일 내에 이미지 및 문자열 같은 정보들을 저장할 수 있습니다.
- 비손실 압축
- 8비트(256색상) 컬러 이미지를 처리 할 수 있습니다.
- 여러장의 이미지를 한 개의 파일에 담을 수 있습니다.(움짤, 애니메이션..)

PNG
GIF의 대체 포맷으로 개발되었습니다.
- 비손실 압축
- 8비트(256 색상) 24비트 (약 1600만 색상) 컬러 이미지를 처리 할 수 있습니다.
- W3C 권장 포맷★
- 알파 채널 지원(투명도)★

WEBP
JPG, PNG, GIF를 모두 대체할 수 있는 구글이 개발한 이미지 포맷입니다.
- 완벽한 손실/ 비손실 압축을 지원합니다.
- GIF 같은 애니메이션 지원
- 알파 채널 지원(손실, 비손실 모두)
- 완벽한 포맷
caniuse.com	<-웹에 필요한 속성, 기능이 사용가능한지 확인.

*파일 경로 작성 방법
1. 절대 경로
고유한 경로입니다. http:// https:// 또는 file:// 의 전체 경로를 사용합니다.
2. 상대 경로
이미지를 삽입할 HTML 문서를 기준으로 경로를 사용합니다.
-이미지가 HTML 문서와 같은 폴더 안에 있는 경우 -> 파일명만 명시
-이미지가 하위폴더에 있는 경우 -> 하위 폴더명/파일명을 명시
-이미지가 상위폴더에 있는 경우 -> ../파일명을 명시	(./은 현재 폴더를 의미)
-이미지가 상위폴더의 다른폴더 안, 다른폴더 안에 있는 경우 -> ../다른폴더명/하위폴더명/파일명을 명시
